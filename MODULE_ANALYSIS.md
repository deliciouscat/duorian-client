# 기존 도구 분석
`Duorian` 라이브러리를 만들기 전, 기존의 정적 web parsing 라이브러리를 철저히 검토하고, 부족한 부분을 발견한다. (동적으로 페이지를 전개하는 과정은 이미 실행되어 있음을 가정한다. 'Chrome extention 사용자가 전개 해놓은' 기반.)

- 검색 및 오픈소스 코드 분석을 통해 각 라이브러리의 동작 과정을 분석한다.
- `test-cases/case-*.html` 파일들을 테스트하며 분석한다.

## 분석 대상 라이브러리
- Cheerio
- jsdom
- parse5
- PostHTML
- Rehype/Remark
- Readability.js
- Trafilatura

## 분석 주안점

**case-1**
댓글 내용까지 제대로 parsing이 되는지 확인 필요
- 댓글 내용: "여당대표 정청래를 보라. 야당에게는 악랄하게 하다가도 이재명이 한마디하면 바로 꼬리를 내리며 하던일도 철회한다. 과거 여당(...이하생략)"

**case-2**
`case-1` 페이지의 내용을 동일하게 담은, 새로운 버전의 웹페이지. `case-1`의 내용에 더하여 AI로 분석된 내용 등 부가적 정보가 추가되었으며, parsing이 더 어려운 구조로 변경된 것으로 읽힌다.

**case-3**
LangChain에 대한 Reddit에서의 논의.
메인 발제에 대한 계층적인 댓글-대댓글 구조로 이루어져 있다.

**case-4**
기아 셀토스에 대한 시승 리뷰

**case-5**
유튜브 동영상 페이지. 페이지의 메인 컨텐츠와 관련된 내용(영상 제목/개요/댓글 등)과 무관한 내용(추천된 동영상, 광고 등)을 분리하는 것이 중요하다.
(물론 메인 컨텐츠 유관성 판별 작업은 server의 작업.)
